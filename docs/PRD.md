## Bootstrap CLI Tool Project Requirements Document

---

#### Description:

The purpose of this project is to develop a tool that can help scaffold a new project in minutes using a CLI. The need for this project was born from the amount of time spent thinking about and implementing initial basic architectures for personal projects which wanted to align with industry standards.Additionally, I wanted to learn more about CLI development and catering towards multiple different OS types.

#### Goals:

1.  Choose a scripting language which can be used on Windows, Mac and Linux to run the bootstrap tool.

2.  Begin implementing individual commands that can generate individual parts of a software architecture or scaffold

3.  Chain the small commands into one command that is capable of creating a full project scaffold following the rules of a industry standard architecture

4.  Design a CLI tool as well as a flow that can guide a user through a singular architecture setup

5.  Continue development, adding more architectures, design patterns, commands and CLI options to flesh out the project.

#### Scope and Deliverables:

1.  A functioning CLI tool that can auto-generate a full project architecture

2.  The ability to customise the project scaffold after creation, which will include:

#### V 1.0 Plan:

- Basic CLI Tool
- Ability to generate a full DDD-style scaffold
- Ability to generate individual DDD-style sub-folders
- Linting, pre-commit checks and Docker options

#### V 1.1 Plan:

- Add extra configurability for full auto-generation (Linting style? pre commit hooks?)
